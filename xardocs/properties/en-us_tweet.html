Description

The Twitter Tweet dynamic data property allows you to easily add a Share, Follow, Hash tag or Mention twitter button on your site for any page or item and customise it with a number of standard Facebook button attributes.

On output, the Twitter button will display the chosen button type, in addition to customisable options available for each button type.

There is no value saved for this Dynamic Data property and no input page. All customisation is in the configuration of the property itself or passed into the tag.

Attributes

  • tweettype: [default='twitter-share-button']
    Determines the style of the button. Options are 'twitter-share-button', 'twitter-follow-button', 'button_hashtag', or 'screen_name'.
  • tweetname: [default='']
    Used in Follow (twitter-follow-button) and Mention (screen_name) buttons and is the twitter name required for the follow or mention button.eg the @follow name.
  • recommend: [default='']
    Used in Follow (twitter-follow-button) and Mention (screen_name) buttons and is the twitter name required for via or mention attribute. eg via #username.
  • showcount: [default='none']
    By default no count is shown. Other options are 'horizontal' or 'vertical'. These show a count of tweets or shares, and either a veritical or horizontal count box for the share button.
  • largebutton: [default=small]
    When true displays a large button.
  • tweettext: [default= current page or article/item title]
    The text used in the tweet.
  • hashtag: [default= '']
    Hashtag to use in Hash tag (button_hashtag) button.
  • tweeturl: [default= the current page url]
    The URL that will be used in the tweet. When left empty it defaults to the current page url.
  • counturl: [default= the tweeturl]
    The count URL that will be used in the tweet if dfferent to the tweet url. Used to overtly specify the url to be counted (the tweet url might be something else or short url).

Other attributes may be inherited from the parent property.

Usage

Standalone

  • In your template forms: you can use the data-output tag to display the Twitter button. 
        <xar:data-display type="tweet" layoutstyle="twitter-follow-button" 
            [attr1="value1",...] />

Defined Data Property

  1. Defining your data property: You can define the attributes in the Configuration screen. To display the property independently, use the data-display tag anywhere you wish the property to display or hook dynamic data to your module/itemtypes for automatic display. 
        <xar:data-input property="$properties['tweet']"
            tweettype="twitter-follow-button" showcount="true" [attr1="value1",...] />
